What YOU must do in the consoles. The code is already done (see end).

WHY: Memberships must be Auto-Renewable Subscriptions (they renew). Coin packs
stay Consumable (unchanged). The app + backend code is updated; it only works
once the store products are subscriptions and the renewal notifications point
at our new endpoints.

PRODUCT-ID SCHEME (important):
  - iOS / Apple : auto-renewable subscription IDs are PREFIXED  subscription_<id>
                  (the old consumable IDs cannot be reused as subscriptions).
  - Android / Google : keep the ORIGINAL unprefixed IDs.
  The app queries the right ID per platform and the backend normalizes the
  `subscription_` prefix, so both map to the same tier/price internally.

APPLE — App Store Connect  (https://appstoreconnect.apple.com)
-------------------------------------------------------------------------------
1. App Store Connect -> your app -> Subscriptions.

2. Group "GreenGo VIP" — add as Auto-Renewable (use the Apple product IDs in Table 1):
     SILVER   : subscription_1_month_silver , subscription_1_year_silver
     GOLD     : subscription_1_month_gold   , subscription_1_year_gold
     PLATINUM : subscription_1_month_platinum , subscription_1_year_platinum_membership
   One group lets users upgrade/downgrade. Set ranking (Platinum highest);
   add localized name, price (Table 1), and review screenshot.

3. Group "GreenGo Base" — one Auto-Renewable yearly product:
     subscription_greengo_base_membership   ($4.99/yr)

4. In-App Purchase Key (backend needs it for the App Store Server API):
   Users and Access -> Integrations -> In-App Purchase -> generate a key.
   Send me the ISSUER ID, KEY ID, and the .p8 file (secret — share securely).

5. APPLE_APP_ID: your app -> App Information -> "Apple ID" (numeric).
   Send it; I set it in functions/.env (APPLE_APP_ID=...).

6. App Store Server Notifications V2 (App Information -> App Store Server
   Notifications): set Production AND Sandbox URL to the Apple endpoint in
   Table 3. Version: Version 2.

-------------------------------------------------------------------------------
GOOGLE — Play Console  (https://play.google.com/console)
-------------------------------------------------------------------------------
1. Monetize -> Products -> Subscriptions. Create one subscription per tier with
   monthly + yearly base plans, using the Google product IDs in Table 1
   (1_month_silver, 1_year_silver, 1_month_gold, ...). Base = its own yearly sub.
   If Play forces a different ID scheme, send me the final IDs and I'll update
   PRODUCT_CONFIG (functions) + the app's ID list.

2. API access / service account (needed to read renewal expiry):
   Setup -> API access -> link the GCP project "greengo-chat". Grant the Firebase
   service account (greengo-chat@appspot.gserviceaccount.com):
     "View financial data, orders, and cancellation survey responses"
     "Manage orders and subscriptions"
   Confirm to me when done.

3. Real-time Developer Notifications (RTDN):
   Monetize -> Monetization setup -> RTDN. Topic: projects/greengo-chat/topics/play-rtdn
   - Create that Pub/Sub topic if it doesn't exist.
   - Grant google-play-developer-notifications@system.gserviceaccount.com the
     "Pub/Sub Publisher" role on the topic.
   - Create a PUSH subscription delivering to the Google endpoint in Table 3.
   - Use "Send test notification" to verify.

WHAT I (the code) ALREADY DID  — committed, NOT deployed yet
-------------------------------------------------------------------------------
APP (Flutter, ships when you build/submit a new app version):
  - Memberships purchase via buyNonConsumable (correct for subscriptions);
    coins still buyConsumable.
  - On iOS the app queries the subscription_<id> products; Android uses the
    originals. Base-membership detection is prefix-agnostic.

BACKEND (functions/src):
  - verifyPurchase normalizes the subscription_ prefix, stores the subscription
    identity (originalTransactionId / purchaseToken) + store expiry so renewals
    can be matched to the user.
  - NEW endpoints:
      appStoreNotificationsV2   (App Store Server Notifications V2)
      playStoreNotifications    (Google Play RTDN via Pub/Sub push)
    They verify the signed payload, find the user, and on renewal extend
    membershipEndDate to the store's real expiry; on cancel keep access until
    expiry; on refund/revoke/expire downgrade to BASIC. Base membership renews
    independently of the VIP tier (never clobbers a higher active tier).
