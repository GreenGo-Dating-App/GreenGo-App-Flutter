WHAT YOU MUST DO  (only iOS — Android needs nothing)
==============================================================================

STEP 1 — Enable Push on the App ID  (Apple Developer)
------------------------------------------------------------------------------
  Certificates, Identifiers & Profiles -> Identifiers ->
    com.greengochat.greengochatapp (GreenGo)
  - Tick the "Enable" checkbox next to "Push Notifications".
  - Do NOT click Configure / Broadcast Capability.
  - Do NOT create an APNs SSL certificate (Certificates (0)) — use the .p8 key.
  - Click Save.

STEP 2 — Create the APNs Authentication Key (.p8)  (Apple Developer)
------------------------------------------------------------------------------
  Certificates, Identifiers & Profiles -> Keys -> (+) Register a New Key
  - Name: "GreenGo APNs"
  - Tick: "Apple Push Notifications service (APNs)"
  - Continue -> Register -> DOWNLOAD the .p8 file (ONE-TIME download; store safely)
  - Write down the KEY ID (10 chars) shown on the key page.
  NOTE: one .p8 works for BOTH development (sandbox) and production, forever.

STEP 3 — Upload the APNs key to Firebase  (Firebase Console)
------------------------------------------------------------------------------
  Firebase Console -> project greengo-chat -> Project Settings ->
    Cloud Messaging tab -> "Apple app configuration" section ->
    APNs Authentication Key -> Upload
  - Upload the .p8 file
  - Key ID:  (from step 2)
  - Team ID: 9885DQB8RF
  - Save
  (Also confirm an iOS app with bundle com.greengochat.greengochatapp is
   registered under Project Settings -> General. Add it if missing.)

STEP 4 — Refresh the provisioning profile so it includes Push  (one-time)
------------------------------------------------------------------------------
  Codemagic's fetch-signing-files REUSES an existing App Store profile if one
  exists — a stale profile created before Step 1 will NOT contain the push
  entitlement, causing a signing/validation failure or silent no-push.
  - Apple Developer -> Profiles -> delete the existing App Store distribution
    profile for com.greengochat.greengochatapp (if any).
  - Re-run the Codemagic "iOS TestFlight Beta" workflow; it recreates a fresh
    profile that now includes Push Notifications.

STEP 5 — Verify Codemagic env group  (Codemagic, one-time check)
------------------------------------------------------------------------------
  Codemagic -> app settings -> Environment variables, group "firebase_credentials":
    GOOGLE_SERVICE_INFO_PLIST  = base64 of ios GoogleService-Info.plist  (optional but referenced)
    FIREBASE_OPTIONS_DART      = base64 of lib/firebase_options.dart
    GOOGLE_SERVICES_JSON       = base64 of android google-services.json
    GOOGLE_MAPS_API_KEY        = maps key
    CERTIFICATE_PRIVATE_KEY    = signing key
  No NEW variable is needed for push — the APNs key lives in Firebase, not here.

==============================================================================
TROUBLESHOOTING
==============================================================================
  - Build error "provisioning profile doesn't include
    com.apple.developer.aps-environment"
      -> Step 1 (App ID Push) not done, OR Step 4 (stale profile) not refreshed.
  - iOS device gets an FCM token but no notification arrives
      -> Step 2/3: the APNs .p8 was not uploaded to Firebase (most common).
  - Android notification not shown when app killed
      -> ensure payload has a `notification` block (it does) and the device is
         not battery-optimizing the app.
  - The second package "com.greengo.dating" inside google-services.json is a
    stale entry and is harmless (applicationId is com.greengochat.greengochatapp).
